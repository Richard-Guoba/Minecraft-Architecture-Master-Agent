# 课程项目强展示版提交包设计

日期：2026-06-27

## 目标

为《大语言模型与信息决策》课程项目准备一套强展示版提交材料，突出项目的工作量、整体性、创新性和真实可复现性。提交材料围绕现有仓库 `Minecraft-Constructing-Agents` 展开，不再制作课堂 PPT，因为课堂展示已经完成。

项目叙事主线为：将中文自然语言建筑需求转化为 Minecraft Java 1.21 可执行数据包。系统不让大模型直接输出方块坐标，而是让大模型负责高层语义 JSON，本地确定性几何引擎负责 CSG 外壳、BSP 房间切分、A* 门洞/楼梯连通、室内装饰落块和 Minecraft 命令导出。

## 课程要求对齐

课程 PDF 的关键约束：

- 书面报告使用 Word 或 LaTeX 排版并转为 PDF，15 页以下。
- 报告以中文为主，简明扼要，逻辑清晰，不在正文粘贴大量代码。
- 如实汇报结果，不伪造运行表现。
- 提交成品以便评判工作量，可提供 GitHub 链接或源码附件。
- 评分重点为工作量、整体性、创新性；性能表现不是主要评分依据。
- 如有必要，报告中说明哪些工作由自己完成，哪些工作借助 AI。

本提交包的应对策略：

- 报告控制在 12 到 14 页，保留必要图示和运行证据，不堆砌源代码。
- GitHub README 和 `docs/` 网站承担快速复现、项目展示和成品入口功能。
- 对缺失的真实运行截图保留明确的补图位置，并写出建议补图 prompt 或文件路径，不虚构结果。
- 在报告中单列“工作分工与 AI 辅助说明”，诚实说明 AI 辅助范围和人工主导内容。

正文结构：

1. 摘要
   - 用 1 页以内说明项目问题、方案和结果。
2. 选题动机
   - Minecraft 建筑生成是开放、多约束、可视化强的任务。
   - 直接让 LLM 输出坐标容易不稳定、不可验证、难复现。
3. 系统总体方案
   - 混合架构：LLM 语义智能体 + 本地确定性几何引擎。
   - 说明为什么把 LLM 控制在 JSON 语义层。
4. 多智能体流程
   - ArchitectAgent、PlannerAgent、MaterialPaletteAgent、CreativeDesignAgent、Structure/Facade/Roof/Site、Interior/Decorator、Repair/QA/Optimizer。
   - 图示从中文 prompt 到 datapack 的数据流。
5. 几何与 Minecraft 导出
   - CSG、BSP、A*、方块目录校验、命令压缩、数据包结构。
6. 模板语料学习与审美闭环
   - 64 个本地 schematic 模板。
   - 案例库、语义条款、设计法则、检索索引、候选择优。
   - 强调模板作为设计语法，不做 1:1 复制。
7. 项目迭代与构思修改
   - 依据 git 历史写 5 到 6 个阶段：
     - 初始 Minecraft architect agent。
     - 自动安装世界与建造模式。
     - 拆分蓝图生成子 Agent。
     - 引入语义规划层和多 Agent 架构。
     - 改造为 construction_method_v1：LLM JSON + 本地几何。
     - 加入模板语料、设计法则、审美审计、自动修复和候选择优。
   - 突出每次修改背后的问题和取舍。
8. 实验与运行结果
   - 现有测试：`npm test`，173 项通过。
   - 展示一到两个本地输出目录中的 `run_report.md`、`preview.html`、`blueprint.json`、datapack。
   - 真实截图如未准备好，则保留“运行结果截图补充位”，并给出补图说明。
9. 项目边界与不足
   - 不是 Mineflayer 实时机器人。
   - 当前以 datapack 导出为主，不自动下载 Minecraft。
   - 自动评分不能完全替代玩家视角审美检查。
10. 工作分工与 AI 辅助说明
    - 人工完成：选题、需求定义、项目方向取舍、运行调试、结果确认、最终提交整合。
    - AI 辅助：代码生成/修改建议、文档初稿整理、报告润色、测试与结构检查辅助。
11. 结论
    - 总结项目完整性、创新点和后续方向。

## 风险与处理

- 风险：报告过长。
  - 处理：正文只放关键图示和结论，把代码细节放 GitHub。
- 风险：截图不足。

  - 处理：采用工程展示风格，强调可复现和评分点。
- 风险：AI 辅助说明影响观感。
  - 处理：如实说明 AI 是辅助工具，突出项目方向、系统取舍、运行验证和整合由小组完成。
