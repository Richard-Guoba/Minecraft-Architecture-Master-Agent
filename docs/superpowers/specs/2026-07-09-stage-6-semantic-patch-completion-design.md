# Stage 6 Semantic Patch Completion Design

Date: 2026-07-09

## Summary

Stage 6 starts with a safe, artifacts-first semantic voxel patch layer. The MVP builds a small deterministic patch dataset from Template Knowledge Base v2 cases and Stage 5 labels, then uses retrieval-style completion to propose local roof, facade, interior, and courtyard patches. It does not yet train a neural network or mutate the main Minecraft build output by default.

The goal is to create the data contract and runtime-safe completion boundary that a later neural patch model can replace. The first implementation is intentionally deterministic so tests, mock mode, and Windows-only Node.js workflows stay stable.

## Current Context

The repository already has the right upstream data:

- `case_library.v2.json` contains cases with style, typology, tags, knowledge units, review state, risk controls, and retrieval tokens.
- `neural_labels.jsonl` contains advisory Stage 5 label suggestions and learning areas.
- `TemplateKnowledgeAgent`, `ExplainableTemplateRetriever`, and `NeuralTemplateRetriever` already preserve rule-only fallback behavior.
- Generation remains Node.js ESM and must work without Python, credentials, or live model calls.

The missing Stage 6 boundary is a local patch representation: small semantic voxel patterns that can be selected, repaired, inspected, and later generated by a trained model.

## Goals

1. Define a `semantic voxel patch` schema with stable source and schema version fields.
2. Build a deterministic patch dataset from v2 knowledge cases and Stage 5 label records.
3. Cover four Stage 6 patch categories: `roof`, `facade`, `interior`, and `courtyard`.
4. Provide a completion API that selects the best patch for a category and context.
5. Add rule post-processing that removes blocked roles, respects clearance boxes, and reports repairs.
6. Keep the first Stage 6 code offline, testable, and disconnected from datapack emission by default.

## Non-Goals

- Do not train a neural model in this MVP.
- Do not require Python for normal tests or generation.
- Do not parse raw schematic block arrays for patch extraction in the first slice.
- Do not let patch completion directly alter CSG output, room layout, paths, or commands yet.
- Do not use a live LLM or embedding API during runtime generation.
- Do not bypass Template KB review/risk controls.

## Architecture

```text
case_library.v2.json
neural_labels.jsonl
        |
        v
buildSemanticVoxelPatchDataset()
        |
        v
stage6 semantic patch dataset
        |
        v
completeSemanticVoxelPatch()
        |
        +--> score candidate patches
        +--> repairSemanticVoxelPatchConflicts()
        +--> return selected patch, semantic voxels, repair report
```

The MVP has two focused modules:

- `src/construction/templates/templateSemanticPatchDataset.js`
  - Owns patch schema, dataset construction, patch JSONL rendering, and artifact writing.

- `src/construction/templates/templateSemanticPatchCompleter.js`
  - Owns patch retrieval, context scoring, deterministic completion, and conflict repair.

## Patch Schema

Dataset:

```json
{
  "source": "stage6-semantic-voxel-patch-dataset-v1",
  "schema_version": 1,
  "generated_at": "2026-07-09T00:00:00.000Z",
  "patch_count": 4,
  "categories": ["courtyard", "facade", "interior", "roof"],
  "patches": []
}
```

Patch:

```json
{
  "patch_id": "house-modern-lake-villa:facade:large-glass",
  "case_id": "house-modern-lake-villa",
  "category": "facade",
  "title": "Modern Lake Villa facade patch",
  "style_family": "modern",
  "typology": "house",
  "tags": ["large-glass", "water-edge"],
  "dimensions": { "width": 5, "height": 5, "depth": 2 },
  "anchor": "facade-plane",
  "semantic_voxels": [
    {
      "x": 2,
      "y": 2,
      "z": 0,
      "role": "view-glass",
      "material_role": "transparent",
      "occupancy": "replaceable",
      "confidence": 0.82,
      "evidence": ["facade knowledge unit"]
    }
  ],
  "risk_controls": ["change exact dimensions and detail placement"]
}
```

Rules:

- `category` must be one of `roof`, `facade`, `interior`, or `courtyard`.
- `semantic_voxels` are local coordinates inside `dimensions`.
- `occupancy` is one of `solid`, `replaceable`, `air`, `furniture`, or `planting`.
- Every voxel has evidence.
- Dataset creation deduplicates patch ids and sorts output for stable diffs.

## Dataset Construction

The builder reads knowledge units and tags:

- `roof` patches come from roof knowledge units, roof tags, terrace/eaves tokens, or roof-related Stage 5 labels.
- `facade` patches come from facade knowledge units, facade tags, glass/trim/balcony tokens, or Stage 5 labels.
- `interior` patches come from interior knowledge units, room-type tags, furnished/furniture labels, or interior learning areas.
- `courtyard` patches come from site knowledge units and site tags such as `courtyard`, `garden`, or `water-edge`.

Each generated patch is a semantic pattern, not a copied structure. Dimensions are small fixed patch windows that capture intent, for example a 5x5 facade bay or a 5x3 courtyard edge.

## Completion And Repair

`completeSemanticVoxelPatch({ dataset, category, context, constraints })` returns:

```json
{
  "source": "stage6-semantic-patch-completer-v1",
  "active": true,
  "mode": "retrieval-completion",
  "category": "facade",
  "selected_patch_id": "house-modern-lake-villa:facade:large-glass",
  "score": 92,
  "semantic_voxels": [],
  "repairs": [],
  "warnings": []
}
```

Scoring uses:

- category match as a hard filter.
- style and typology match.
- requested tags from prompt/context.
- prompt tokens such as glass, garden, roof, interior, courtyard, water, terrace.
- patch confidence and review risk penalty.

Repair rules:

- Drop voxels with roles in `constraints.blocked_roles`.
- Drop solid or furniture voxels that intersect `constraints.clearance_boxes`.
- Drop voxels outside patch dimensions.
- Keep `air` voxels when they mark a reserved circulation or opening space.
- Return every repair as a structured record with reason and voxel role.

## Error Handling

- Empty dataset: return inactive completion with warning.
- Unknown category: return inactive completion with warning.
- No candidate patch: return inactive completion with warning.
- Invalid patch voxels: remove invalid voxels and record repairs.
- All voxels removed by constraints: return active selection with empty `semantic_voxels` and a warning so callers can fall back to rules.

## Testing Strategy

Unit tests cover:

- Dataset builder creates all four patch categories from a mixed knowledge-base fixture.
- Patch records include stable ids, dimensions, semantic voxels, tags, and evidence.
- JSONL rendering is stable and parseable.
- Completion selects a style/tag-matching facade patch over an unrelated candidate.
- Completion repair removes blocked roles and clearance collisions while preserving safe opening/air cells.
- Empty or unknown category inputs return fallback-safe inactive results.

Integration with the main construction pipeline is deferred until the patch contract is stable and inspected.

## Success Criteria

- Stage 6 has a committed design and an executable plan.
- Tests prove four patch categories can be derived from existing Template KB v2 style data.
- Tests prove a deterministic completer can select and repair a patch without changing generation output.
- Full test suite remains green.
- The Stage 6 boundary is ready for later raw schematic patch extraction or neural model training.

## Deferred Work

- Raw schematic patch extraction from NBT block arrays.
- A Python or PyTorch local training subsystem.
- Provider-assisted voxel completion.
- Runtime integration into facade, roof, site, and interior agents.
- Visual patch gallery for human review.
