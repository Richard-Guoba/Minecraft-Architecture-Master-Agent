Minecraft Architecture Master Agent

A long-term hybrid architecture agent that turns natural-language building requests into runnable Minecraft Java datapacks through semantic planning, deterministic geometry, template memory, benchmarking, and repair loops.

Built As A Hybrid Agent, Not A Block Matrix Guess

The language model produces high-level semantic JSON. Local JavaScript owns the geometry, rooms, paths, stairs, furniture, validation, scorecards, and datapack export. This keeps the creative layer flexible while keeping Minecraft output reproducible.

  1. 01Prompt understandingStyle, typology, site, materials, and constraints.
  2. 02Semantic planningMassing, room graph, facade, roof, site, and interior intent.
  3. 03Deterministic constructionCSG shell, BSP rooms, A* doors and stairs, template-aware site modules.
  4. 04Evaluation and exportQA, repair hints, scorecard, preview, report, mcfunction, datapack.

Current Capabilities

Construction Pipeline

Generates legal Minecraft Java 1.21 datapacks with shell, roof, windows, rooms, entry approach, stairs, interiors, site modules, preview, report, and scorecard.

Template Memory v2

Converts local schematics into reviewed case cards, knowledge units, retrieval indexes, design laws, and explainable prompt references.

Readable Benchmarks

Runs a fixed 10-prompt Architecture Master baseline and writes JSON, CSV, Markdown, gallery, and human feedback templates.

Repair Readiness

Tracks weak scorecard dimensions and repairs them through tests, generation changes, and rerun baseline evidence.

Stage 1 Readiness Baseline

The current main branch has an empty repair queue. The baseline is not the final ambition; it is the stable floor for bigger work like multi-concept design and critic-driven repair.
