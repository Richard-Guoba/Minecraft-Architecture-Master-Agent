《大语言模型与信息决策》课程项目

从中文建房需求到 Minecraft 可执行数据包

本项目构建了一套 LLM 语义智能体 + 本地确定性几何引擎的建筑生成流水线。LLM 不直接输出坐标，而是生成高层 JSON；坐标、房间、门洞、楼梯、装饰和命令由本地程序生成并校验。

系统架构

项目没有把 prompt 直接翻译成 Minecraft 方块坐标，而是把不稳定的语言理解限制在语义层，再用确定性程序完成几何、连通和命令导出。这是项目最核心的设计取舍。

LLM 负责什么

  • ArchitectAgent 生成风格、材料、体块和外壳规则。
  • PlannerAgent 生成房间拓扑、楼层功能和连通关系。
  • CreativeDesignAgent 选择体块变化、屋顶语言和场地策略。

本地程序负责什么

  • CSGBuilder 生成空心外壳和体块网格。
  • BSPPartitioner 切分室内房间。
  • AStarPathfinder 打通门洞和楼梯。
  • Decorator、QA、Optimizer 写入装饰、校验方块并压缩命令。

复现方式

默认 mock 模式不需要 API key，便于课程评阅时快速复现完整输出。

本地运行

npm install
npm test
npm start -- --mode mock "建一个欧式大房子"

Minecraft 中建造

复制 out/<timestamp>/architect_datapack 到世界 datapacks
/reload
/function architect:run

边界与诚实说明

当前版本不是 Mineflayer 实时连服机器人，也不模拟玩家逐块放置；主交付是可复现的数据包生成流水线。
AI 辅助用于代码建议、文档初稿整理和报告润色；项目方向、系统取舍、运行调试、结果确认与最终整合由小组完成。
