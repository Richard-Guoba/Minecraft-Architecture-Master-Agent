Semantic intent. Deterministic Minecraft architecture.

The active construction_method_v1 pipeline turns Chinese or English building requests into Minecraft Java 1.21 datapacks. A separate local workflow trains on all 64 templates, keeps artifacts private by default, and has now passed its first held-out non-air learning gate.

Construction flow

Prompt → semantic agents → deterministic geometry → QA and repair → datapack.

  • Semantic planningStyle, massing, rooms, circulation, facade, roof, structure, and site intent.
  • Local geometryCSG shells, BSP rooms, A* paths, interiors, decoration, and Minecraft command validation.
  • Reproducible outputMock mode needs no API key and writes datapack, preview, report, and scorecard artifacts.

Local training flow

Local templates → automatic preparation → source-level split → training → evaluation.

  • No per-file approvalAll existing local templates may be used for local experiments.
  • Leakage preventionBuildings and duplicate groups are split before whole volumes or patches are created.
  • Measured progressGate 1 and Gate 2 passed; the architectural-shape class remains the main known weakness.
  • Local by defaultPrepared data, weights, metrics, and reconstructions stay below .local/training/.
